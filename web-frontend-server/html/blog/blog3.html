Monitoring với Prometheus và Grafana

Ngày đăng: 17/11/2025 | Tác giả: Pham Gia Huy & Pham Khanh Linh

Tại sao cần Monitoring?

Trong môi trường production, việc giám sát hệ thống là BẮT BUỘC để:

  • Phát hiện sự cố sớm trước khi ảnh hưởng người dùng
  • Theo dõi hiệu năng và xu hướng tăng trưởng
  • Tối ưu hóa tài nguyên (CPU, RAM, Disk, Network)
  • Phân tích nguyên nhân khi có downtime

Prometheus - Time Series Database

Prometheus là hệ thống monitoring mã nguồn mở, sử dụng mô hình pull-based để thu thập metrics từ các target.

Ưu điểm:
Cấu hình đơn giản qua YAML
PromQL - ngôn ngữ query mạnh mẽ
Service Discovery tự động
Alerting tích hợp sẵn

Grafana - Data Visualization

Grafana giúp trực quan hóa dữ liệu từ Prometheus (và nhiều datasource khác) thành các dashboard đẹp mắt, dễ hiểu.

Metrics quan trọng cần theo dõi

  1. CPU Usage: Nếu > 80% liên tục → cần scale up
  2. Memory Usage: RAM free < 10% → nguy cơ OOM
  3. Disk I/O: Latency cao → database sẽ chậm
  4. Network Traffic: Bandwidth usage và packet loss
  5. Response Time: API latency phải < 200ms

Demo trong dự án MiniCloud

Trong hệ thống của chúng tôi, Prometheus thu thập metrics từ:

  • Node Exporter → Hardware/OS metrics
  • Web Frontend Server → HTTP request metrics
  • Docker containers → Container stats

Grafana hiển thị dashboard "System Health" với 3 panel chính: CPU, Memory, Network Traffic.

Best Practice: Thiết lập alerting rule để nhận thông báo qua Email/Slack khi có bất thường!

Kết luận: Prometheus + Grafana là combo hoàn hảo cho monitoring trong môi trường Cloud và Container.
